## 1. Market Context & User Demand (Non-Technical)
### 1.1 Why Super-Agents Are Emerging Now
AI agents are transitioning from passive chat assistants into active operators capable of executing real tasks on devices. Users are demanding:
- Reliable task execution
- Real-world automation
- Personalized memory
- Proactivity without annoyance
- A single interface to simplify their digital lives

### 1.2 What Users Hate About Current AI Agents
- Limited device control
- Forgetfulness
- Hallucinations
- Notification spam
- Slow responses
- Overly formal or robotic tone
- Poor integration with phone apps
- Dependency on cloud-only models

### 1.3 Opportunity Window
No existing product offers:
- On-device automation + memory + personality + agent orchestration
- Cross-app task execution
- Secure hybrid offline capability
- Clear reasoning and transparency

This creates a large opportunity for differentiation.

#### 4.2.1 Device Interaction Layer

#### 4.2.2 Memory Core
- Vector store
- Structured JSON memory

#### 4.2.3 Orchestration Engine
- High-level intent parser
- Tool router
- State manager
- Safety layer
- Autonomous task scheduler

#### 4.2.4 Model Stack
- Local small model for:
  - summarization
  - offline Q&A
  - classification
  - voice commands
- Cloud large model for:
  - planning
  - reasoning
  - multi-step workflows

### 4.3 Data Flow
1. User input → Intent Parser
2. Intent → Planner LLM
3. Planner → Action Graph
4. Action Graph → Device Tools
5. Execution results → Post-processor
6. Memory storage
